## Directory Purposes

**src/app/:**
- Purpose: Next.js App Router - all routable pages and API endpoints
- Contains: Page components (.tsx), layouts, API route handlers (.ts)
- Key files: `layout.tsx`, `page.tsx`, `api/` folder

**src/app/(auth)/:**
- Purpose: Public authentication pages
- Contains: Login and registration pages
- Key files: `login/page.tsx`, `register/page.tsx`

**src/app/(dashboard)/:**
- Purpose: Protected dashboard pages (requires authentication)
- Contains: Dashboard overview, invoice list, invoice detail, upload page
- Key files: `layout.tsx` (wraps with Sidebar and SessionProvider), `dashboard/page.tsx`, `invoices/page.tsx`, `invoices/upload/page.tsx`, `invoices/[id]/page.tsx`

**src/app/api/:**
- Purpose: REST API endpoints
- Contains: HTTP handlers for auth, invoices, and operations
- Key files:
  - `auth/[...nextauth]/route.ts` - NextAuth callback
  - `invoices/route.ts` - GET (list), POST (upload)
  - `invoices/[id]/route.ts` - GET (detail), PATCH (update)
  - `invoices/[id]/approve/route.ts` - POST (approval)
  - `register/route.ts` - POST (user registration)

**src/components/ui/:**
- Purpose: Base UI components from shadcn/ui design system
- Contains: Unstyled but accessible components (Button, Card, Dialog, Input, etc.)
- Pattern: Wrapper components around Radix UI primitives with Tailwind styling
- Key files: `button.tsx`, `card.tsx`, `input.tsx`, `badge.tsx`, `dialog.tsx`, `table.tsx`

**src/components/invoices/:**
- Purpose: Feature-specific components for invoice operations
- Contains: Upload form, IVA classification badges
- Key files: `invoice-upload-form.tsx`, `iva-badge.tsx`

**src/components/layout/:**
- Purpose: Structural layout components
- Contains: Sidebar with navigation, global layout structure
- Key files: `sidebar.tsx`

**src/components/providers/:**
- Purpose: React context providers and wrappers
- Contains: SessionProvider (wraps app with NextAuth context)
- Key files: `session-provider.tsx`

**src/lib/ai/:**
- Purpose: OpenAI integration and invoice extraction pipeline
- Contains: API calls, prompt engineering, response parsing
- Key files:
  - `extract-invoice.ts` - Main extraction function (uses gpt-4o vision)
  - `prompts.ts` - System and user prompts for extraction
  - `parse-response.ts` - Zod schema validation of AI response

**src/lib/tax/:**
- Purpose: Domain logic for Uruguayan tax system
- Contains: IVA classification, RUT validation, tax calculations
- Key files:
  - `iva-classifier.ts` - Classification logic (3-tier strategy)
  - `iva-categories.ts` - Tax rate constants
  - `rut-validator.ts` - RUT format validation

**src/lib/auth.ts:**
- Purpose: NextAuth configuration and credentials provider
- Contains: JWT strategy, credentials verification, password hashing
- Key: Centralized auth setup used by all protected routes

**src/lib/db.ts:**
- Purpose: Prisma database client initialization
- Contains: PostgreSQL adapter setup, singleton pattern
- Key: Single import point for all database access

**src/types/:**
- Purpose: Shared TypeScript interfaces and type definitions
- Contains: Invoice extraction result, classified line items, tax constants
- Key files: `invoice.ts` (ExtractionResult, ClassifiedLineItem), `tax.ts` (IVA_RATES)

**src/generated/prisma/:**
- Purpose: Auto-generated Prisma types and models
- Contains: Prisma client, enums (InvoiceStatus, IvaCategory, UserRole, etc.), type definitions
- Generated by: `prisma generate` command
- Note: Do NOT edit manually - regenerate after schema changes

**prisma/:**
- Purpose: ORM configuration and schema
- Contains: Schema definitions for all models (User, Invoice, Company, Vendor, etc.)
- Key files: `schema.prisma` (models and relations)

**public/:**
- Purpose: Static assets served directly
- Contains: Images, favicons, fonts
- No TypeScript/JSX files here

## Naming Conventions

**Files:**

- Page components: kebab-case with `/page.tsx` suffix in directory
  - Example: `invoices/page.tsx`, `invoices/upload/page.tsx`, `invoices/[id]/page.tsx`
- API route handlers: `/route.ts` suffix in directory
  - Example: `api/invoices/route.ts`, `api/auth/[...nextauth]/route.ts`
- Components: PascalCase.tsx
  - Example: `InvoiceUploadForm.tsx`, `SessionProvider.tsx`, `Button.tsx`
- Utilities: kebab-case.ts
  - Example: `extract-invoice.ts`, `iva-classifier.ts`, `rut-validator.ts`
- Configuration: kebab-case or implicit names
  - Example: `auth.ts`, `db.ts`, `prompts.ts`

**Directories:**

- Feature directories: kebab-case or plural for collections
  - Examples: `invoices/`, `api/`, `components/`, `lib/`
- Layout groups: (parentheses)
  - Examples: `(auth)`, `(dashboard)`
- Dynamic segments: [brackets]
  - Examples: `[id]`, `[...nextauth]`

**Variables & Functions:**

- camelCase for functions and variables
  - Examples: `extractInvoiceFromBase64()`, `classifyIva()`, `handleSubmit()`
- PascalCase for React components
  - Examples: `InvoiceUploadForm`, `SessionProvider`, `DashboardPage`
- UPPER_SNAKE_CASE for constants
  - Examples: `EXTRACTION_SYSTEM_PROMPT`, `IVA_RATES`

**Types & Interfaces:**

- PascalCase for type/interface names
  - Examples: `ExtractionResult`, `ClassificationResult`, `DashboardStats`
- Enum PascalCase, members PascalCase or UPPER_CASE
  - Examples: `enum InvoiceStatus { PENDING, PROCESSING, EXTRACTED, ... }`

## Where to Add New Code

**New Invoice Processing Feature:**
- Primary code: `src/app/api/invoices/route.ts` or new `/[action]/route.ts`
- Tax logic: `src/lib/tax/`
- Types: `src/types/invoice.ts`
- Frontend: `src/components/invoices/` and `src/app/(dashboard)/invoices/`

**New Dashboard Page:**
- Page component: `src/app/(dashboard)/[feature-name]/page.tsx`
- Specific components: `src/components/[feature-name]/`
- API route: `src/app/api/[feature-name]/route.ts`

**New Utility/Helper:**
- General utilities: `src/lib/utils.ts`
- Domain-specific: `src/lib/[domain]/[feature].ts`
  - Examples: `src/lib/tax/`, `src/lib/ai/`

**New Component:**
- Feature components: `src/components/[feature]/ComponentName.tsx`
- Base UI components: Already in `src/components/ui/` (use shadcn/ui pattern)
- Layout/structural: `src/components/layout/`

**New Type Definition:**
- Domain types: `src/types/[domain].ts`
- All invoice-related: `src/types/invoice.ts`
- All tax-related: `src/types/tax.ts`

## Special Directories

**node_modules/:**
- Purpose: Installed npm dependencies
- Generated: Yes (by npm install)
- Committed: No (ignored by .gitignore)
- Note: Do not edit manually

**.next/:**
- Purpose: Next.js build output and development cache
- Generated: Yes (by next dev/build)
- Committed: No (ignored by .gitignore)
- Note: Do not edit manually

**src/generated/prisma/:**
- Purpose: Prisma client and type definitions
- Generated: Yes (by prisma generate)
- Committed: No (ignored by .gitignore in most projects, may be committed for type checking)
- Note: Regenerate after schema changes with `npm run db:push`

**.git/:**
- Purpose: Git version control repository
- Generated: Yes (by git init)
- Committed: Yes (git metadata)
- Note: Do not edit manually

**.env, .env.local, .env.production:**
- Purpose: Environment variables (secrets and configuration)
- Generated: No (created by developers)
- Committed: No (ignored by .gitignore)
- Note: Must be created locally for development and deployment

**prisma/:**
- Purpose: Prisma configuration and migrations
- Contains: schema.prisma (source of truth), seed.ts, migrations/ (if using file-based migrations)
- Key: Essential for database management
