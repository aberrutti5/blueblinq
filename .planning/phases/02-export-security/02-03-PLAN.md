Add immediately after the `if (!file)` null check (before any `file.arrayBuffer()` call):

```typescript
const ALLOWED_TYPES = [
  "image/jpeg",
  "image/png",
  "image/webp",
  "application/pdf",
] as const;

if (!ALLOWED_TYPES.includes(file.type as (typeof ALLOWED_TYPES)[number])) {
  return NextResponse.json(
    { error: "Tipo de archivo no permitido. Usá JPG, PNG, WebP o PDF." },
    { status: 400 }
  );
}
```

Define `ALLOWED_TYPES` as a `const` array at module level (outside the handler function) so it's not recreated on every request. The check uses `file.type` which is the MIME type from the multipart form data Content-Type header — this is server-side and cannot be bypassed by client-side code modifications.

No other changes to the file. No new imports needed. Error message in Spanish consistent with existing error style.
