Rewrite the PATCH handler in `src/app/api/invoices/[id]/route.ts` to verify invoice ownership before updating.

The fixed PATCH handler must:
1. Validate session — return 401 if missing (already done, keep).
2. Get `userId` from session: `(session.user as { id: string }).id`.
3. Fetch the user's `companyId` via `db.companyMembership.findFirst({ where: { userId, isDefault: true }, select: { companyId: true } })` — return 400 if null.
4. Fetch the invoice: `db.invoice.findUnique({ where: { id }, select: { companyId: true } })`.
5. If invoice not found → return 404 `{ error: "Factura no encontrada" }`.
6. If `invoice.companyId !== companyId` → return 403 `{ error: "No autorizado" }`.
7. Only then perform `db.invoice.update({ where: { id }, data: body, include: { lineItems: true } })` and return result.

Also add the same companyId check to the GET handler — it currently returns any invoice by ID without verifying ownership. Apply the same pattern: fetch invoice, check companyId, return 403 if mismatch.

Add `import type { NextRequest } from "next/server"` is already present. No new npm dependencies. Error messages in Spanish.
